## [3.2.0] - 2026-05-19

### 新增 / Added

  - `global()` 输出显示函数定义（如 `area(r) = 3.14*r*r`）
- 📑 **函数列表标签页**：新增"函数"Tab，分三个区域展示
  - 局部函数：当前会话中定义的函数
  - 🌐 全局函数：跨会话共享的函数

### 改进 / Improved
- 📝 `process_text()` 新增 `preset_functions` 参数，支持预设函数注入
- 💾 会话持久化格式新增 `global_functions` 字段，向后兼容

## [3.1.2] - 2026-05-19

### 修复 / Fixed
- 🛠️ 修复更新重启时闪现命令行窗口（ping/timeout 窗口）的问题
  - Windows：改用 PowerShell `-WindowStyle Hidden` 实现完全无窗口的延迟等待和重启
  - macOS：使用 `open` 命令启动新进程，等待旧进程退出后再启动
  - Linux：使用 `start_new_session` + 后台进程，等待旧进程退出后再启动
  - 所有平台：清除 `_MEIPASS2` 环境变量，清理 `.old` 残留文件

## [3.1.1] - 2026-05-19

### 修复 / Fixed
- 🛠️ 修复更新后重启报错 `Failed to load Python DLL _MEI*\python39.dll` 的问题
  - 根本原因：子进程继承了旧进程的 `_MEIPASS2` 环境变量，导致新 exe 去已删除的 `_MEI` 临时目录查找 DLL
  - 修复：启动新进程前清除 `_MEIPASS2`/`_MEIPASS` 环境变量，延迟增加到 5 秒，使用 `start` 命令启动独立进程

## [3.1.0] - 2026-05-19

### 新增 / Added
- 🧮 **自定义函数**：支持用户定义带参数的函数，并在后续表达式中调用
  - 定义语法：`func(x, y) = 3*x + 5*y`
  - 调用语法：`func(1, 2)` → 结果为 23
  - 支持隐式乘法：`f(x) = 3x + 1` 等价于 `f(x) = 3*x + 1`
  - 支持嵌套调用：`quad(x) = double(double(x))`
  - 函数体可引用已定义的变量：`g(x) = x + offset`
  - 支持表达式作为参数：`func(3+2, 4*2)`
  - 参数数量校验，错误时给出明确提示
  - 语法高亮支持函数定义

## [3.0.3] - 2026-05-19

- 🛠️ 修复 `global()` 声明的变量在其他标签页中仍报"变量未定义"的问题

  - 大端字节序：MSB（最高位）从索引 0 开始，从左到右 0→31
  - 小端字节序：LSB（最低位）从索引 0 开始，显示 31→0（从左到右）
  - 位值右对齐，与索引对齐

## [1.1] - 2026-03-05

### 重大变更 / Breaking Changes
- 🔄 **bitmap 语法重构**：将 `endian:` 命令和 `bitmap` 关键字合并为函数形式
  - 旧语法：`endian: big` + `bitmap 变量 = 值`
  - 新语法：`bitmap(值, 1)` 或 `变量 = bitmap(值, 1)`
  - `bitmap(值)` 或 `bitmap(值, 0)` 表示小端字节序（默认）
  - `bitmap(值, 1)` 表示大端字节序
  - 函数无返回值，只能单独使用或配合变量赋值

### 新增 / Added
- ✨ 新增 `bitmap()` 函数，支持参数化的字节序控制
- 📚 新增文档：`bitmap函数说明.md` 和 `bitmap-Function-Guide.md`

### 改进 / Improved
- 🎯 简化了位结构查看的语法，更符合函数式编程习惯
- 📖 更新了所有示例代码以使用新的 bitmap 函数语法
- 🔧 优化了代码结构，移除了全局字节序状态

### 移除 / Removed
- ❌ 移除了 `endian:` 命令（被 bitmap 函数参数替代）
- ❌ 移除了 `bitmap` 关键字语法（改为函数调用）
